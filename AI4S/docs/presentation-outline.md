## 三、7.6.4 迭代搜索 ⭐（1~1.5分钟重点）

### 先展示框架图

```
第1轮: 基础扩展(110万) → SIS → 拟合 → 选top-3种子
                        ↓
第2轮: 种子 × 原始特征交叉(仅270个) → SIS → 拟合 → 新种子
                        ↓
                     …直到收敛 → 输出最优公式
```

### 一句话讲设计
> "核心思路：第1轮先跑标准SISSO得到高分种子。第2轮**不再盲目扩110万**，而是用种子和原始特征定向交叉，每轮只产生270个新特征。"

### 展示收敛路径

| 轮次 | 操作 | R² | 发现了什么 |
|------|------|-----|-----------|
| 1 | 标准SISSO | 0.8731 | `α^(1/3)·γ/(kT·R)` — 缺Ω |
| **2** | **种子×Ω** | **0.9429** | **`α^(1/3)·γ·Ω/(kT·R)` — 完整结构 ✅** |
| 3 | 继续交叉 | 0.9403 | 尝试Ω^(4/3)，回落 |
| 4 | 继续交叉 | 0.9429 | 确认收敛 |

### 关键结论
> "第2轮最关键：缺Ω的种子和原始Ω做交叉——**等效实现了三元组合**，R²从0.873跳到0.943，公式结构与目标完全一致。而全空间搜索做不到这一点，因为它只做一次二元扩展。"

### 三种策略对比（一句话）
> "试了三种引导策略：**layered**（保留原始特征+种子交叉）最好，R²=0.9429；**replace**（丢弃原始特征）只有0.873。"

## 五、TA问答准备

| 可能的问题 | 回答要点 |
|-----------|---------|
| **为什么迭代比全量好？** | 全量只做一次二元扩展，无法生成三元结构。迭代通过多轮种子交叉等效实现了三元组合。 |
| **怎么选种子的？** | 每轮SIS筛选后取top-3公式，用它们和原始5特征的幂次变体做乘除交叉。 |
| **seed×seed想过吗？** | 试过，但种子高度同质化（都是同一公式），效果不如seed×原始特征。 |
| **为什么R²不是1？** | 结构已经正确，剩余5.7%是数据噪声。 |
