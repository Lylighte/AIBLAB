## 一、模型结构

### 1.1 网络设计

```
输入: 梅尔频谱 (batch, time_steps, 64)
    ↓
4层 RNN (hidden_size=256, batch_first=True, dropout=0.2)
    ↓
取最后一个时间步输出 (batch, 256)
    ↓
Batch Normalization (256)
    ↓
Dropout (rate=0.2)
    ↓
全连接层 (256 → num_classes=35)
    ↓
输出: 35个命令词的分类概率
```

### 1.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `input_size` | 64 | 梅尔频谱的 Mel 维度 |
| `hidden_size` | 256 | RNN 隐藏状态维度 |
| `num_layers` | 4 | RNN 层数 |
| `dropout_rate` | 0.2 | Dropout 比率 |
| `batch_size` | 256 | 每批样本数 |
| `learning_rate` | 0.001 | Adam 优化器学习率 |
| `num_epochs` | 10 | 训练轮数 |
| `n_mels` | 64 | 梅尔滤波器组数 |
| `n_fft` | 1024 | FFT 窗口大小 |
| `hop_length` | 512 | 帧移 |

### 1.3 TODO 补全代码（核心部分）

#### TODO 1: RNN 层

```python
self.rnn = nn.RNN(input_size, hidden_size, num_layers, batch_first=True, dropout=dropout_rate)
```

- `batch_first=True`：输入形状为 `(batch, seq_len, features)`，与 `collate_fn` 中 `pad_sequence` 的 `batch_first=True` 保持一致
- `dropout=dropout_rate`：多层 RNN 层间加入 Dropout 防止过拟合

#### TODO 2: 全连接层

```python
self.fc = nn.Linear(hidden_size, num_classes)
```

- 将 RNN 最后一个时间步输出的 `hidden_size` 维向量映射到 `num_classes`（35个命令词类别）

### 1.4 数据处理流程

1. **音频加载**：使用 `soundfile` 读取 `.wav` 文件（16kHz，单声道）
2. **特征提取**：`torchaudio.transforms.MelSpectrogram` 将波形转为梅尔频谱
   - 输出形状：`(1, 64, T)` → 压缩后 `(T, 64)`
3. **序列填充**：`pad_sequence` 将同 batch 内不同长度的序列填充到统一长度
4. **标签编码**：将命令词字符串映射为数字索引

## 三、结果分析

### 3.1 模型表现

1. **准确率趋势**：整体呈上升趋势，从第 1 轮 11.0% 提升到第 10 轮 23.2%（验证集），最终测试集 **25.2%**
2. **收敛速度**：Loss 从 3.25 下降到 2.77，说明模型在学习但收敛较慢
3. **波动问题**：第 5 轮和第 8 轮准确率出现回落，说明基础 RNN 存在梯度不稳定问题

### 3.2 基础 RNN 的局限性

| 问题 | 说明 |
|------|------|
| **梯度消失** | RNN 在处理长序列时，梯度在反向传播中指数级衰减，导致早期时间步的信息难以被学习 |
| **长距离依赖** | 4 层 RNN 堆叠后，BPTT 路径更长，梯度消失更严重 |
| **拟合能力有限** | 相比 GRU/LSTM，基础 RNN 没有门控机制，无法有效选择记忆或遗忘信息 |

### 3.3 参数选择说明

| 参数 | 选择理由 |
|------|----------|
| `hidden_size=256` | 平衡模型容量和计算开销，SpeechCommands 数据量不大，256 足够 |
| `num_layers=4` | 多层 RNN 能提取更高层次的时序特征 |
| `dropout=0.2` | 防止过拟合，因为训练集只有 ~10000 条 |
| `batch_size=256` | RTX 4070 显存充裕（12GB），大 batch 加速训练 |
| `lr=0.001` | Adam 优化器的默认学习率，经实践验证效果稳定 |

### 4.2 思考题（实验报告要求 1,2,3,6）

详见 `Doc/RNN.md` 第 9.7 节），以下为简要回答：

**1. 为什么不直接处理语音，而是使用梅尔谱或 MFCC 特征？**

原始语音波形采样率高、数据量大，且包含大量冗余信息。梅尔谱/MFCC 模拟人耳听觉特性，在保留语音关键特征的同时大幅降维，减少模型参数量和计算量，有助于模型更快收敛。

**2. 语音和文本是不等长的，代码中是如何处理 batch 训练的？**

采用 `pad_sequence` 将同 batch 内的所有序列填充到最大长度，填充值为 0。RNN 处理时通过 `batch_first=True` 对齐维度，但填充位置仍参与计算——这也是为什么实验中在 RNN 后只取最后一个时间步的输出，而不是对所有时间步做 pooling。

**3. 端到端语音识别实验使用的字典里一共有多少个汉字？是怎么得到的？**

THCHS-30 数据集的字典包含约 2000-3000 个常用汉字，通过对训练集文本标注进行词频统计得到，覆盖了数据集中出现的所有汉字。

**6. 在 ASR 代码里，哪处地方避免了连续两个"天"字被合并？**

在使用 CTC（Connectionist Temporal Classification）损失函数时，CTC 解码中的"去冗余合并"规则会合并连续的相同字符，而空白符（blank）会分隔相同字符。通过在处理时保留空白符位置，避免了"天天"被错误合并为"天"。

## 五、进阶方向（9.5.4）

若需达到 **80% 以上**，建议将 `nn.RNN` 替换为 `nn.GRU` 或 `nn.LSTM`：

```python
# GRU 版本
self.rnn = nn.GRU(input_size, hidden_size, num_layers, batch_first=True, dropout=dropout_rate)

# LSTM 版本
self.rnn = nn.LSTM(input_size, hidden_size, num_layers, batch_first=True, dropout=dropout_rate)
```

GRU/LSTM 通过门控机制解决梯度消失问题，能更好地捕捉语音中的长时序依赖，显著提升分类准确率。
