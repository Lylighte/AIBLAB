## 一、模型结构

### 1.1 网络设计

```
输入: 梅尔频谱 (batch, time_steps, 64)
    ↓
4层 GRU (hidden_size=256, batch_first=True, dropout=0.2)
    ↓
取最后一个时间步输出 (batch, 256)
    ↓
Batch Normalization (256)
    ↓
Dropout (rate=0.2)
    ↓
全连接层 (256 → num_classes=35)
    ↓
输出: 35个命令词的分类概率
```

### 1.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `input_size` | 64 | 梅尔频谱的 Mel 维度 |
| `hidden_size` | 256 | GRU 隐藏状态维度 |
| `num_layers` | 4 | GRU 层数 |
| `dropout_rate` | 0.2 | Dropout 比率 |
| `batch_size` | 256 | 每批样本数 |
| `learning_rate` | 0.001 | Adam 优化器学习率 |
| `num_epochs` | 10 → 20 | 训练轮数（优化版增至 20） |
| `scheduler` | CosineAnnealingLR | 学习率余弦退火调度（优化版） |
| `n_mels` | 64 | 梅尔滤波器组数 |
| `n_fft` | 1024 | FFT 窗口大小 |
| `hop_length` | 512 | 帧移 |

### 1.4 数据处理流程

1. **音频加载**：使用 `soundfile` 读取 `.wav` 文件（16kHz，单声道）
2. **特征提取**：`torchaudio.transforms.MelSpectrogram` 将波形转为梅尔频谱
   - 输出形状：`(1, 64, T)` → 压缩后 `(T, 64)`
3. **序列填充**：`pad_sequence` 将同 batch 内不同长度的序列填充到统一长度
4. **标签编码**：将命令词字符串映射为数字索引

## 三、结果分析

### 3.1 RNN 基础版分析

1. **准确率趋势**：整体呈上升趋势，从第 1 轮 11.0% 提升到第 10 轮 23.2%（验证集），最终测试集 **25.2%**
2. **收敛速度**：Loss 从 3.25 下降到 2.77，说明模型在学习但收敛较慢
3. **波动问题**：第 5 轮和第 8 轮准确率明显回落，说明基础 RNN 存在**梯度消失**问题

### 3.2 GRU 进阶版分析

1. **显著提升**：仅换一行代码（RNN → GRU），测试准确率从 **25.2% → 79.2%**，提升 54%
2. **快速收敛**：第 1 轮验证准确率即达 31.0%（RNN 为 11.0%），第 3 轮已超 60%
3. **Loss 持续下降**：从 2.91 降至 0.45，说明 GRU 有效缓解了梯度消失，学习能力远强于 RNN
4. **接近目标**：79.2% 仅差 0.8%，增加训练轮数或微调参数即可达到 80%+

### 3.3 调参优化版分析

1. **成功突破 80%**：增加轮数至 20 + CosineAnnealingLR 调度器，测试准确率达 **81.2%** ✅
2. **Loss 大幅下降**：从 GRU 10 轮的 0.45 降至 **0.084**，模型拟合更充分
3. **学习率自动衰减**：CosineAnnealingLR 从 0.001 平滑降至接近 0，后期精细调参
4. **验证集稳定超 80%**：第 13 轮起验证准确率稳定在 80% 左右，最高 80.8%

### 3.4 RNN vs GRU 对比分析

| 对比维度 | RNN | GRU |
|---------|-----|-----|
| **核心机制** | 简单循环，无门控 | 重置门 + 更新门 |
| **梯度传播** | 易梯度消失/爆炸 | 门控缓解梯度问题 |
| **长时序建模** | 弱（有效记忆短） | 强（可捕获长依赖） |
| **参数量** | 少 | 略多（约 3 倍） |
| **计算速度** | 快 | 几乎相同 |
| **本实验准确率** | 25.2% | 79.2% → **81.2%** |

### 3.5 参数选择说明

| 参数 | 选择理由 |
|------|----------|
| `hidden_size=256` | 平衡模型容量和计算开销，SpeechCommands 数据量不大，256 足够 |
| `num_layers=4` | 多层 GRU 能提取更高层次的时序特征 |
| `dropout=0.2` | 防止过拟合，训练集约 10000 条，适中 Dropout 保持泛化 |
| `batch_size=256` | RTX 4070 显存充裕（12GB），大 batch 加速训练 |
| `lr=0.001` | Adam 优化器默认学习率，经实践验证效果稳定 |

### 4.2 思考题（实验报告要求 1,2,3,6）

### 4.2 思考题（实验报告要求 1,2,3,6）

详见 `Doc/RNN.md` 第 9.7 节），以下为简要回答：

**1. 为什么不直接处理语音，而是使用梅尔谱或 MFCC 特征？**

原始语音波形采样率高、数据量大，且包含大量冗余信息。梅尔谱/MFCC 模拟人耳听觉特性，在保留语音关键特征的同时大幅降维，减少模型参数量和计算量，有助于模型更快收敛。

**2. 语音和文本是不等长的，代码中是如何处理 batch 训练的？**

采用 `pad_sequence` 将同 batch 内的所有序列填充到最大长度，填充值为 0。RNN 处理时通过 `batch_first=True` 对齐维度，但填充位置仍参与计算——这也是为什么实验中在 RNN 后只取最后一个时间步的输出，而不是对所有时间步做 pooling。

**3. 端到端语音识别实验使用的字典里一共有多少个汉字？是怎么得到的？**

THCHS-30 数据集的字典包含约 2000-3000 个常用汉字，通过对训练集文本标注进行词频统计得到，覆盖了数据集中出现的所有汉字。

**6. 在 ASR 代码里，哪处地方避免了连续两个"天"字被合并？**

在使用 CTC（Connectionist Temporal Classification）损失函数时，CTC 解码中的"去冗余合并"规则会合并连续的相同字符，而空白符（blank）会分隔相同字符。通过在处理时保留空白符位置，避免了"天天"被错误合并为"天"。

通过增加轮数 + 余弦退火学习率调度器，成功突破 **80%**，达成 **81.2%** ✅

| 方法 | 改动 | 效果 |
|------|------|------|
| **增加轮数** | `num_epochs=10` → `20` | Loss 从 0.45 降至 0.084 |
| **CosineAnnealingLR** | 学习率从 0.001 平滑衰减至接近 0 | 后期精细调参，稳定超 80% |
| **综合提升** | 轮数 + 调度器 | Test Acc **79.2% → 81.2%** ✅ |
